(Via drafts-eval@iana.org): IESG/Authors/WG Chairs:

The IANA Functions Operator has completed its review of draft-mcgrew-hash-sigs-13. If any part of this review is inaccurate, please let us know.

We identified a possible issue with the document's description of the new registries' registration procedure.

The document states, "IANA SHOULD verify that all applications for additions to these registries have first been reviewed by the IRTF Crypto Forum Research Group (CFRG)."

We believe that if IANA is meant to perform this function, "SHOULD" should be changed to "must"/"MUST", as IANA can't choose whether to perform this verification. However, it may be appropriate to incorporate this into the designated experts' review, as IANA can't evaluate whether there were any issues with the mailing list review (if this is relevant). (If the Specification Required procedure is going to be used, one or more experts will need to be designated to review the specification. We do have confirmation that the IESG can designate experts for this document.)

We understand that IANA will create the following registries at https://www.iana.org/assignments/xmss-extended-hash-based-signatures:
