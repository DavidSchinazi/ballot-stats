Abstract


  This document adds a new capability called "SAVEDATE" to the Internet
  Message Access Protocol (IMAP).  It defines a new IMAP message
  attribute called 'save date' that, unlike the existing 'internal
  date' attribute, always indicates the moment at which the message was
  saved in its current mailbox.  The SAVEDATE capability extends the
  FETCH command with the means to retrieve the save date attribute and
  it extends the SEARCH command to allow using that attribute in
  searching criteria.

Technical Summary

  This spec extends IMAP with an additional data point on each
  email in a mailbox which records the time at which the email
  was added to the mailbox.

Working Group Summary

  The only topic of debate was how to deal with mailboxes which don't
  support storing the savedate.  Various ideas were debated, and a
  conclusion was found which everybody was happy with.

Document Quality

  This specification has been implemented in multiple servers and is
  very easy to follow.
