(Via drafts-eval@iana.org): IESG/Authors/RFC ISE:

The IANA Services Operator has completed its review of draft-pantos-http-live-streaming-21. If any part of this review is inaccurate, please let us know.

We have one question about this document.

When the RFC Independent Stream Editor notifies us that the actions requested in this document can be performed, we will replace the current template for media type registration application/vnd.apple.mpegurl with the template contained in this document.

There is one issue, though: in the future, we may create a "file extension" column at https://www.iana.org/assignments/media-types, which would be populated by extracting data from the "file extension" fields in registration templates.

Can the file extension information in this template be placed in a field labeled "File extension(s):", as in RFC 6838?

Thank you,

Amanda Baber
Lead IANA Services Specialist
