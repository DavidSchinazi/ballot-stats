[Ballot comment]
The extensions defined in this document are "an optional TRILL switch capability".  To me, that indicates that the base TRILL specifications rfc6325 and rfc7177 (in this case) are not affected: an RBridge is TRILL-compliant as long as it implements what rfc6325 specifies (without these optional extensions).  I would then like to see the formal "Updates" tags removed.

[The publication of this document is not the place to argue about the meaning of "Updates", so I'll defer to what the Responsible AD decides.]

The IANA Services Operator understands that, upon approval of this document, there is a single action which we must complete.

In the PORT-TRILL-VER Sub-TLV Capability Flags regsitry on the Transparent Interconnection of Lots of Links (TRILL) Parameters registry page located at:

https://www.iana.org/assignments/trill-parameters/

a single registration consisting of two adjacent bits will be made as follows:

Bit: [ TBD-at-Registration ] and [ TBD-at-Registration ]
Description: Topology labeling support
Reference: [ RFC-to-be ]

We understand that these two bits are to be adjacent and from the bits available between 14 and 31. IANA also notes that the authors suggest bits 14 and 15 for this assignment.

The IANA Services Operator understands that this is the only action required to be completed upon approval of this document.

Note:  The actions requested in this document will not be completed until the document has been approved for publication as an RFC. This message is meant only to confirm the list of actions that will be performed.

Please note that specific values cannot be reserved. However, early allocation is available for some types of registrations. For more information, please see RFC 7120.

Thank you,

Sabrina Tanamal
Senior IANA Services Specialist

The IESG plans to make a decision in the next few weeks, and solicits final

sent to iesg@ietf.org instead. In either case, please retain the beginning of
the Subject line to allow automated sorting.

Abstract


  This document specifies extensions to the IETF TRILL (Transparent
  Interconnection of Lots of Links) protocol to support multi-topology
  routing of unicast and multi-destination traffic based on IS-IS
  (Intermediate System to Intermediate System) multi-topology specified
  in RFC 5120. This document updates RFC 6325 and RFC 7177.
