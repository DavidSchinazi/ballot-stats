# Technical Summary

  This document addresses the first of the questions raised
  in RFC 9217. It does so by specifying a set of path properties
  which might be useful to undertake various path-related tasks
  by involved entities. Examples of such tasks are: selecting
  between paths or invoking some of the services that are
  supported on a given path.

# Working Group Summary

  This document ran 4 versions before adoption by the RG. No contentious
  issues were encountered during the development of the document.

  Regular slots were dedicated to this draft in several RG meetings.
  The authors implemented the outcomes of the discussions held in these
  meetings and the mailing list. They also implemented some post-RGLC
  comments to better align with some relevant work in
  the IETF (e.g. ALTO, TEAS).
 
  The document benefited from various reviews, which helped to enhance
  the quality of the document.

  The Shepherd reviewed this draft several times (prior adoption,
  after adoption, etc.). These detailed reviews can be found, e.g., at:

  * https://github.com/boucadair/IETF-Drafts-Reviews/raw/master/draft-enghardt-panrg-path-properties-02-rev%20Med.pdf
  * https://github.com/boucadair/IETF-Drafts-Reviews/raw/master/draft-enghardt-panrg-path-properties-03-rev%20Med.pdf 

  The draft was also updated to address the comments raised by Colin Perkins, mainly:

  * https://github.com/panrg/path-properties/pull/82/files (Add two statements described in RFC 5743)

  * https://github.com/panrg/path-properties/pull/83 (clarify that paths can be selected by the network or the endpoints)

  * https://github.com/panrg/path-properties/pull/84 (fairness, privacy, and proxy discussion)

  -08 was released to address the IRSG review from Dave Oran. FWIW, the full set of issues/resolutions can be found at:

  * https://github.com/panrg/path-properties/issues?q=irsg+review

# Personnel

  Mohamed Boucadair is the Document Shepherd.
  Colin Perkins is the IRTF Chair.

# Intellectual Property

  Each author has confirmed conformance with BCP 78/79. There are no IPR
  disclosures on the document.

# Technical Summary

  This document addresses the first of the questions raised
  in RFC 9217. It does so by specifying a set of path properties
  which might be useful to undertake various path-related tasks
  by involved entities. Examples of such tasks are: selecting
  between paths or invoking some of the services that are
  supported on a given path.

# Working Group Summary

  This document ran 4 versions before adoption by the RG. No contentious
  issues were encountered during the development of the document.

  Regular slots were dedicated to this draft in several RG meetings.
  The authors implemented the outcomes of the discussions held in these
  meetings and the mailing list. They also implemented some post-RGLC
  comments to better align with some relevant work in
  the IETF (e.g. ALTO, TEAS).
 
  The document benefited from various reviews, which helped to enhance
  the quality of the document.

  The Shepherd reviewed this draft several times (prior adoption,
  after adoption, etc.). These detailed reviews can be found, e.g., at:

  * https://github.com/boucadair/IETF-Drafts-Reviews/raw/master/draft-enghardt-panrg-path-properties-02-rev%20Med.pdf
  * https://github.com/boucadair/IETF-Drafts-Reviews/raw/master/draft-enghardt-panrg-path-properties-03-rev%20Med.pdf 

  The draft was also updated to address the comments raised by Colin Perkins, mainly:

  * https://github.com/panrg/path-properties/pull/82/files (Add two statements described in RFC 5743)

  * https://github.com/panrg/path-properties/pull/83 (clarify that paths can be selected by the network or the endpoints)

  * https://github.com/panrg/path-properties/pull/84 (fairness, privacy, and proxy discussion)
