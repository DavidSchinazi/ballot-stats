(Via drafts-lastcall@iana.org): IESG/Authors/WG Chairs:

The IANA Services Operator has completed its review of draft-ietf-grow-mrt-add-paths-03.txt. If any part of this review is inaccurate, please let us know.

The IANA Services Operator understands that, upon approval of this document, there are three actions which we must complete.

First, in the BGP4MP_ET Subtype Codes subregistry of the Multi-threaded Routing Toolkit (MRT) registry located at:

https://www.iana.org/assignments/mrt/mrt.xhtml#BGP4MP_ET-codes

four new codes are to be registered as follows:

Value: 8
Name: BG4MP_MESSAGE_ADDPATH
Reference: [ RFC-to-be ]

Value: 9
Name: BG4MP_MESSAGE_ADDPATH
Reference: [ RFC-to-be ]

Value: 10
Name: BG4MP_MESSAGE_ADDPATH
Reference: [ RFC-to-be ]

Value: 11
Name: BG4MP_MESSAGE_ADDPATH
Reference: [ RFC-to-be ]

Second, in the BGP4MP Subtype Codes subregistry of the Multi-threaded Routing Toolkit (MRT) registry also located at:

https://www.iana.org/assignments/mrt/mrt.xhtml#BGP4MP-codes

four new codes are to be registered as follows:

Value: 8
Name: BG4MP_MESSAGE_ADDPATH
Reference: [ RFC-to-be ]

Value: 9
Name: BG4MP_MESSAGE_ADDPATH
Reference: [ RFC-to-be ]

Value: 10
Name: BG4MP_MESSAGE_ADDPATH
Reference: [ RFC-to-be ]

Value: 11
Name: BG4MP_MESSAGE_ADDPATH
Reference: [ RFC-to-be ]

Third, in the TABLE_DUMP_V2 Subtype Codes subregistry of the Multi-threaded Routing Toolkit (MRT) registry also located at:

https://www.iana.org/assignments/mrt/mrt.xhtml#table-dump-v2-subtype-codes

five new codes are to be registered as follows:

Value: 8
Name: RIB_IPV4_UNICAST_ADDPATH
Reference: [ RFC-to-be ]

Value: 9
Name: RIB_IPV4_MULTICAST_ADDPATH
Reference: [ RFC-to-be ]

Value: 10
Name: RIB_IPV6_UNICAST_ADDPATH
Reference: [ RFC-to-be ]

Value: 11
Name: RIB_IPV6_MULTICAST_ADDPATH
Reference: [ RFC-to-be ]

Value: 12
Name: RIB_GENERIC_ADDPATH
Reference: [ RFC-to-be ]

The IANA Services Operator understands that these three actions are the only ones required to be completed upon approval of this document.

Note:  The actions requested in this document will not be completed until the document has been approved for publication as an RFC. This message is only to confirm what actions will be performed.


Thank you,

Sabrina Tanamal

From: The IESG
To: "IETF-Announce"

Subject: Last Call:  (Multi-Threaded Routing Toolkit (MRT) Routing Information Export Format with BGP Additional Paths Extensions) to Proposed Standard


The IESG has received a request from the Global Routing Operations WG
(grow) to consider the following document:
- 'Multi-Threaded Routing Toolkit (MRT) Routing Information Export
Format
  with BGP Additional Paths Extensions'
  as Proposed Standard

The IESG plans to make a decision in the next few weeks, and solicits
final comments on this action. Please send substantive comments to the
ietf@ietf.org mailing lists by 2017-02-15. Exceptionally, comments may be
sent to iesg@ietf.org instead. In either case, please retain the
beginning of the Subject line to allow automated sorting.

Abstract


  This document updates the Multi-threaded Routing Toolkit (MRT) export
  format for Border Gateway Protocol (BGP) routing information by
  extending it to support the Advertisement of Multiple Paths in BGP
  extensions.




The file can be obtained via
https://datatracker.ietf.org/doc/draft-ietf-grow-mrt-add-paths/

IESG discussion can be tracked via
https://datatracker.ietf.org/doc/draft-ietf-grow-mrt-add-paths/ballot/


No IPR declarations have been submitted directly on this I-D.
